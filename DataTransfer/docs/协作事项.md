# 数据传递微服务迭代协作事项

> 适用范围：`SmartFactory-2026/DataTransfer`
>
> 用途：记录开发当前迭代时，为了保持迭代边界清晰而转入未来迭代的事项。这里不记录普通想法或长期愿望，只记录会影响后续实现、验收、跨服务对齐的待办。

## 维护规则

1. 每个事项都必须写明来源迭代、目标迭代、前置关联与完成标准。
2. 事项进入本文档后，必须在目标迭代内完成；若依赖其他微服务先完成接口或业务决策，则状态标为“外部前置”，并保留对应的 CSC 编号或对接对象。
3. 最终迭代收尾时，应逐条检查本文档。除仍被外部微服务阻塞的事项外，不应留下未完成项。
4. 若未来实现发现某事项不再需要，应把状态改为“已取消”，并写明取消依据；不要直接删除。
5. 本文档只记录 DataTransfer 自身视角的开发协作事项。跨服务总体约束仍以仓库根目录的 `跨服务协作备忘.md` 为准。

## 状态说明

| 状态 | 含义 |
|------|------|
| 待办 | 已明确需要在未来迭代实现，尚未开始 |
| 进行中 | 当前迭代正在实现 |
| 已完成 | 已实现、已测试，并且完成标准全部满足 |
| 等待确认 | 已明确未来处理方向，但需要外部环境、上游能力或业务决策确认后再进入实现 |
| 外部前置 | 需要其他微服务先完成接口、语义或业务决策 |
| 已取消 | 经确认后不再需要实现，需保留取消原因 |

## 当前已知边界

P0 已完成 Go + Protobuf + gRPC/MQTT 的运行骨架，包含配置加载、管理端点、内存消息流、近期消息拉取、基础指标、`command_id` 去重和 P0 指令拒绝回执。

P1 已完成 Modbus TCP 静态 Connector、首个 Converter、Command Router、Ack Tracker、内部 DiscoveryEvent 日志与指标，以及启动 smoke 测试。
